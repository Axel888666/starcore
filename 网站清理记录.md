## 项目结构优化后

### 根目录结构
```
starcore/
├── content/          # 页面内容
├── layouts/          # 页面布局
├── static/           # 静态资源
├── public/           # 生成的网站
├── themes/           # 主题文件
├── RCS/              # 参考文件
├── i18n/             # 国际化文件
├── archetypes/       # 内容模板
├── data/             # 数据文件
├── assets/           # 资源文件
├── hugo.toml         # Hugo配置
├── README.md         # 项目说明
├── 项目说明.md        # 详细文档
├── build.bat         # 构建脚本
├── serve.bat         # 开发脚本
└── deploy.bat        # 部署脚本
```

### 主要目录说明
- **content/** - 包含所有页面的markdown内容
- **layouts/** - 包含所有页面的HTML布局模板
- **static/** - 包含CSS、图片等静态资源
- **themes/event/** - 当前使用的主题
- **RCS/** - 保留的参考HTML文件

## 清理原则

### 删除标准 ✅
1. **临时文件** - 开发过程中的临时记录文档
2. **重复文件** - 不使用的主题和压缩包
3. **空目录** - 没有内容的空目录
4. **无用文件** - 不再需要的配置或资源文件

### 保留标准 ✅
1. **核心功能** - 网站运行必需的文件
2. **配置文件** - Hugo和项目配置
3. **内容文件** - 所有页面内容和布局
4. **资源文件** - 图片、CSS等静态资源
5. **文档文件** - 重要的项目说明文档
